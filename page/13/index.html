IoT: Arduino and HTTP made simple

Posted on April 9, 2014 by lasote

Today I bring you a new block (lasote/arduino_http) to make simpler the interaction between Arduino and HTTP.

Interaction between Arduino and HTTP with biicode:

  1. High-level: Forget about writing HTTP headers and CRLF. Make GET and POST requests in an easy way.
  2. Response: Don’t write complex flow for manage timeouts, socket response code or stream availability. Just use the stream processors you need or write your own ones.
  3. Low memory: Through the stream processors the memory consumption remains low.
  4. No hardware dependent: You can use your favorite ethernet or wifi adapter.
  5. Extensible: You can add custom HTTP headers and write your own stream processors. So you can program your own classes to interact with a concrete web service.

Introducing the open command

Posted on March 25, 2014 by julia

Today we are introducing the open command. If you’ve already played a bit with biicode you have seen how easy is to reuse code that other people have published: You just write the #include directive (or require, or import… dependending on your programming language), and the code gets downloaded into your deps folder.

Now we are going to talk about how to continue improving code that you have already published. There are two possible scenarios here: If you still have the hive where your published your block from, you can keep editing it there and carry on publishing new versions. But what if you formatted your computer, or want to publish from Bahamas and you didn’t bring your laptop with you? Here is where the open command comes in handy.
