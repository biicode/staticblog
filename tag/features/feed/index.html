]]> Biicode is not a Version Control System, neither a Source Code Management System. As a dependency manager, biicode makes “some kind” of internal version control to establish dependencies against a specific version of the block.

Our versions are incremental integers starting on 0. Until now,  to know that biicode’s version corresponds to any official version you had to look for it on the block’s description or readme.

Now you can tag a version while publishing. For example, qiangwu/atlstl corresponds to the official “1.9.98” version:

Using Version Tags

Let’s say you’re publishing a new version into biicode and you’re looking for a way to relate this version number to your SCM versions or your library’s official version release. Today, I’m tagging OSCPack library version 3 to the official release version 1.1.0:

~/myproject$ bii publish  --tag STABLE --versiontag=v1.1.0
INFO: Publishing block: Maria/oscpack
INFO: *****************************
INFO: ***** Publishing public ****
INFO: *****************************
INFO: Successfully published Maria/oscpack: 3 @v1.1.0

That’s the basics.

Version Tags Utilities

Publish a new version with its tag:

~/myproject$ bii publish --tag STABLE --versiontag 1.2.0
INFO: *****************************
INFO: ***** Publishing public ****
INFO: *****************************
INFO: Downloading files from: amalulla/hi
INFO: Successfully published amalulla/hi: 2 @1.2.0

Semantic Versioning

A versioning policy is just some simple rules specifying how to use version numbers. It can be very simple like the one biicode makes by default or more complex.

We encourage you developers to follow the Semantic Versioning standard for your block’s versions. Be mindful that  using a “non-rational” policy will be a disservice for those using your blocks.

Depending with Version Tags

To use a version based on a tag instead of biicode internal version number, write in [requirements] section in your biicode.conf:

amalulla/hi @1.2.0

And whenever you build your project, biicode will discover biicode version and update your [requirements] section in your biicode.conf:

amalulla/hi: 2 @1.2.0

Not to confuse them with Tags

Add tags and a description to your block, this is useful for other users and will be indexed better for our Search Engine.

Just select settings and start writing and adding tags!

]]> This is is a small explanation about BITSCAN, the C++ library exclusively developed by Pablo San Segundo.

BITSCAN is dedicated to the efficient processing of bit strings. In programming, a bit string is a data structure that stores collections of 1s and 0s. It gets interesting when those 1s and 0s refer to a Boolean property of a group, so that each element of this group is identified by a bit of that chain.

Time to move your pawns forward

A good example of this is the game of chess. In a chessboard each position is formed by six different types of pieces with two possible colours. A string of 64 bits can encode the position of all pieces of the same type and color on the board (e.g. “white pawns”) by referring each bit to a square with the semantics of a value to 1 (TRUE) if the square is occupied by a piece of the chosen type and 0 (FASLE) otherwise.

The figure below shows an example. The white pawns on the board are encoded the 64-bit string to the right.

Recently bit strings have had much impact in the scientific community because it has been shown that bit-parallel algorithms can accelerate up to one order of magnitude the solution of a number of complex combinatorial problems.
After years of research in combinatorial optimization, Professor San Segundo has developed C++ bit string library BITSCAN optimizing those features that he found useful in his work, including different types of bit scanning.

More on BITSCAN

As you can recall, in a recent comparative BITSCAN was proven clearly more efficient than other C, C++ widespread implementations (i.e. STL bitset and BOOST dynamic_bitset).

Are you interested in scanning bits? Pablo is going to release soon a great surprise for those interested in encoding simple unweighted graphs.

]]> Thanks to your feedback we are improving faster than expected. We are working hard based on the feedback we get from our users. This week’s release brings you an easier way of installing and configuring biicode than ever before.

Download Biicode Installer and install it now!

Installation wizards

There are new installation wizards for C/C++, Arduino and Raspberry Pi. You don’t need to manually download external tools anymore. To set them up you only need to execute one of the following commands:
C/C++ -> bii setup:cpp
Arduino -> bii setup:arduino
Raspberry Pi -> bii setup:rpi

Here’s a screenshot of how it looks like writing bii setup:cpp

 

As it was meant to be, we do all the heavy lifting for you. Look below how it installs and checks everything is installed. In case you are not sure, re-run the command again and see everything is already installed in green.
The same thing happens when you are setting up Arduino or Raspberry Pi.

Settings management wizards

We’ve also created settings wizards so you don’t need to manually edit any configuration files. Whenever you are establishing or changing your settings, just write
bii arduino:settings or bii rpi:settings and answer the wizard’s questions!

Beside this, you don’t need to worry again for which port is your Arduino connected: we automatically detect which port is it in. So the bii arduino: usb command it’s no longer available.

Any doubts? Check our docs for help:

Biicode docs C/C++
Biicode docs Arduino
Biicode docs Raspberry Pi

Happy coding!
