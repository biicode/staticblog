That’s what communicating with a serial device is, or serving applications over HTTP… you’re pushing bits, shoveling bits.. whatever you want to call this.

Now if you need to run some optimized algorithms where cycles seriously count to the tune of changing an application from taking days to taking hours, okay… give me C++. If I need to modify those bits with special CPU instructions or need to know what data lies on L1 cache or whether access is going to suffer from NUMA bottle-necking in practice.. okay, give me C or C++.

But I hope to never write another asynchronous webserver in C++ for as long as I live! Thank the heavens we have a fast viable alternative such as Go, Rust, Nim, DLang, Scala, or even PyPy!
