bii internals: Packaging a compiled app with PyInstaller

Posted on May 23, 2014 by julia

In a previous post we talked about how we compiled biicode into c extensions. Today we are going to talk about how we prepare it for execution with PyInstaler. PyInstaller is a program that converts (packages) Python programs into stand-alone executables, under Windows, Linux, Mac OS X, Solaris and AIX. It does so by packaging together your code with all third party libraries your code is using along with python runtime libraries. Third party libraries are discovered inspecting imports in your code but this is a problem when your code is in binary extension.

Introducing the open command

Posted on March 25, 2014 by julia

Today we are introducing the open command. If you’ve already played a bit with biicode you have seen how easy is to reuse code that other people have published: You just write the #include directive (or require, or import… dependending on your programming language), and the code gets downloaded into your deps folder.

Now we are going to talk about how to continue improving code that you have already published. There are two possible scenarios here: If you still have the hive where your published your block from, you can keep editing it there and carry on publishing new versions. But what if you formatted your computer, or want to publish from Bahamas and you didn’t bring your laptop with you? Here is where the open command comes in handy.

This blog post is out dated.

Project configuration and setup

Project configuration in software development is the process of setting up how your code will be built, managed and/or run. It varies depending on the programming language and tools, as the IDE, used. For instance, when programming in C/C++, the project setup usually comes with defining which artifacts (executables, libraries) will be built from certain source code files, with given compiling and linking options. In Microsoft Visual, this can be mostly done with wizards, menu and contextual commands; e.g. you can configure your directories containing external libraries in a dialog (Project Properties).
