]]> A month later, I finally have enough time to detail the second Using std::cpp day (October 28th) organized at the University Carlos III of Madrid by Jose Daniel García, well-known professor for being the head of the Spanish C++ Standards Committee and member of ISO C++ Standards Committee.

Since C++ is being improved with the new version of its specification (C++11 and beyond), there is a lot of interest in Spain in the new language features’. Using std::cpp has become the Spanish C++ forum where the C++ developers can express their experiences, ideas and start discussions about the present and the future of the language.

Using std::cpp 2014: the talks

The first talk was “Basic functional programming” by the great Joaquín M. Lopez in which he detailed the use of lambda function in C++11, in order to improve the performance and the quality of source code .

In the next talk “Unspecified vs Undefined” Juan Soulié, author of the web site cplusplus.com, defined the grey areas of the language from which any developer should avoid if possible.

After a slight break with a coffee in hand, sponsored by Indizen, Diego Rodriguez Losada, co-founder and CTO of biicode, presented some support systems for developers to develop software easily such, as CMake or NPM for Python. But his last talk was the best, it focused on biicode a deps manager that is developed to solve the classical nightmare of handling dependencies in C/C++. Slides:

Dependencies Managers in C/C++. Using stdcpp 2014 from biicode

Before lunchtime, Jose Daniel started a discussion about a controversial design pattern with “Singleton must die”.

With renewed strength after lunch, Joaquin again presented the talk “At last I understand the monads” used especially for futures. If someone has interest in this topic, do not forget that the next 15h January, Joaquin will present “De 0 a 100 (Taller): mónadas en C++14”.

Folowing this talk, Pablo San Segundo, Assistant professor of UPM and chess master, presented his management library of bits in C++: BITSCAN.

After the last coffee break, sponsored by biicode, “Know or not to Know? Risk management for application development in C ++” we learned the thing that we should take into account for real developing into real world.

Finally, with “Object Calisthenics for C++” with Anibal Cáceres, software developer of Ericsson, we discovered we were not developing properly in C++.

The videos of all talks are in the new youtube channel Using std::cpp [SPANISH].

We have already started preparing the next Using std::cpp meeting, which I hope is at least as good as this last one.
